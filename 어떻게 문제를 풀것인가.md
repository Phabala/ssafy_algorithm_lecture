# 어떻게 문제를 풀 것인가

## 1. 문제 이해
- 조건을 파악한다
- 구하는 것이 무엇인가
- 그림, 도표, 그래프로 표현
- 예전에 비슷한 문제를 풀어본 적 있는가?

## 2. 설계
**설계에 시간을 30~40% 쓰는게 좋다.**
- To Do List
- 시간 복잡도 검증
- 시뮬레이션

## 3. 구현
**설계가 잘 되있고 구현력만 있다면 구현은 크게 신경 쓸 필요 없다. 구현력을 키우는 것은 평소에 많이 풀어보는게 방법이다. 실전에서는 구현에 노력을 쏟으면 안된다.**
- 작명에 신경 쓰기
- 가독성 있게 구현하기

## 4. 디버깅
- 단축키, Stepping 연습 해놓기
- 원인이 되는 지점 추측
